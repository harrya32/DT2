- (A) Knowledge of the candidate policy set

We believe there are a number of real-world applications where (some) knowledge of the policies likely to be unrolled in a DT will be available. E.g. medical settings, where treatments are relatively standardised, etc. But, we do not need complete knowledge of Pi, cf. our cancer experiments, where we show results in decision-making amongst unseen policies, and DT2 still has an improvement here.


- (B) Preference ordering

We did not consider equalities, since in our experiments the rewards are continuous, and therefore two policies will almost surely not have the same value. Nevertheless, we agree that equality should be incorporated for completeness...

As for your question about our theorem, the 'better preference ordering' here should hold for most reasonable comparison metrics, e.g. Spearman's. The spearman's of the L_sim optimal model can trivially be improved upon when we look at the constructions of our proofs.

- (C) DT estimated values

Yes, this is estimated via Monte Carlo simulations, just like the true value is estimated in the real environment. We use 500 rollouts.

- (D) Lambda ablation

We did try setting lambda=1, but it led to quite unstable learning, we think because of the general difficulty of learning to rank (cite some LTR papers?). We see that the simulation loss regularises the loss, and makes it far more consistent learning. Also, in these environments the FQE generally had better regrets/spearmans than the DT2 models, and we did not achieve exactly the same rankings as these proxy ground-truths with lambda less than 1, or equals to 1.

- (E) VaGraM architecture

We used the architecture proposed in the VaGraM paper. Please also now see our response to reviewer YLjT, where we document results to more offline MBRL baselines, where, again, we follow the architectural choices made in their papers.

- (F) Weighted MSE

This is an interesting question! While the loss function does not directly weight the sim loss based on the ranking loss, instead adding them, it does seem that, at least in this experiment, it appears that it could be recast that way. Nevertheless, in general it is not clear how we should decide on this weighting. The ranking loss is almost an automatic way to discover this

- (G) Coverage of pi

We think this is a good question as well. While the unseen policies in cancer experiment are all distinct from those in training, there is undoubtedly some level of 'overlap' between them. We see robust analysis of this question as an area for future work, however, as already mentioned in our limitations section. It would be interesting to see if we can limit the policies that need to be used during training, for efficiency purposes, but also, it would be good if we can provide some guidance on the coverage, and what kind of policies we expect to be well-ranked.
